Project 2 Documentation

What we want:

  • Present the user with their “perfect pokemon”

How will we do this:

  • We need to look at the traits of the pokemon and give the user options to choose the traits they like. Then, we’ll look at the pokemon to see which one matches, if any.

What traits should we focus on?

  • The original idea was to focus on nature and gender, but I’ve been told I should include more options. The PokeAPI gives us the following endpoints:
    • Abilities
    • Pokeathlon stats
    • Locations
    • Colors
    • Forms
    • Habitats
    • Shapes
    • Species
  • I do need to note, however, that as far as I know, I need to know the names of each thing in these categories in order to send the request to the endpoint. Basically, I need to manually type out the names of the shapes, for example, have the user choose a shape name, then send that name to the server, then get the name of the pokemon back.
  • Update: nature is too broad since pokemon do not have natures specific to their species, so we’ll have to use a different attribute. I’ll go with type instead

Okay, let’s get into the details, step by step:

  • First, the user chooses a trait from the dropdown menu (this will work the same for each one)
    • We should make the name of the dropdown the name of the endpoint, so then we can just make a function that makes a generic request based on the name and the item chosen
  • After choosing the items, make the server request for the information
    • If they choose none for one of them, just don’t make a request. Basically, it will go to the endpoint and it won’t request anything OR it just won’t go to the endpoint (which would be ideal)
    • We’ll have to make sure that the chosen pokemon match each of the criteria. My mind is saying that we could get all of each type and compare them all. For example:
      • Gender: female color: blue nature: timid
      • So I think I should find every blue pokemon, see which ones are female, then see which ones are timid
      • I could also do nature first, so I’d find every timid pokemon, then find every timid blue pokemon, then find every timid blue female pokemon
        • This one sounds a bit better because there will probably be less information to parse through (instead of looking at every female pokemon since there are a lot, look at every timid one, and there won’t be as many)
  • The information we display should (ideally) be the following:
    • Pokemon photo (if possible)
    • Pokemon name (of course)
    • Pokemon region
    • Possibly the moves
    • If anything else, maybe some stats, but that can come later

How are we going to display it all?

  • The lower half of the page could and should be dedicated to the results, so we can display each one in a div and just put the character info inside of the div for now, later we can have the info pop up in a separate div on hover or click

What controls will there be?

  • The choice dropdowns (3-5)
  • Search button
  • If we want, we can include a bar to search for names, but that kind of defeats the purpose of the site finding pokemon for you
  • Ideally the option to click on the pokemon name in the results and be taken to a link to the wiki page about the pokemon (we’ll have to look into that later, too)
  • If possible, a reset button that empties out the results area
